WHAT THE PIPELINE DOES:

1. FETCH - Downloads 5,465 institutions from College Scorecard API
   • Handles 1,000 req/hour rate limit
   • Caches JSON locally to ./data_cache/
   • ~55 API calls, ~2-3 minutes

2. TRANSFORM - Normalizes JSON to SQL format
   • Flattens nested structures
   • Normalizes demographics (wide→narrow)
   • Extracts 89K+ field of study programs
   • Exports CSVs for inspection
   • ~2-5 minutes

3. LOAD - Inserts into MySQL with validation
   • Batch inserts (1,000 rows at a time)
   • FK constraint checking
   • Transaction management
   • Summary statistics
   • ~5 minutes

TROUBLESHOOTING:

API errors
  → Check API key in .env
  → Verify internet connection
  → Check rate limit (1000 req/hour)

MySQL connection errors
  → Verify MySQL server is running
  → Check credentials in .env
  → Ensure database name in .env (or use --create-schema)

Foreign key violations
  → Run --create-schema first
  → Verify dimension tables are populated

Memory issues
  → Reduce batch size in db_loader.py
  → Process subsets by state
